# 文档样式规范

**作者**: RJ.Wang
**创建时间**: 2026-04-24

生成学习文档、指南、笔记类 Markdown 文件时，遵循以下规范。

---

## Mermaid 图表规范

### 全局配置
每个 mermaid 代码块必须在第一行加入 init 配置，统一缩小字号和间距：

```
%%{init: {"theme":"default","themeVariables":{"fontSize":"12px"},"flowchart":{"nodeSpacing":15,"rankSpacing":30}}}%%
```

### 布局方向
- 步骤较多时优先使用纵向布局 `graph TD`，避免横向 `graph LR` 导致图太宽、字太小
- 节点内容精简，关键信息写在节点文字里，避免过多 subgraph 嵌套导致图过大
- 用颜色区分不同类别的节点（如绿色=优势、红色=劣势、蓝色=RAG、橙色=微调）

## 批注（学习追问）规范

文档中的学习批注用于记录学习过程中的真实疑问和解答，与正文区分开。

### 结构
使用 HTML `<details>` 折叠块，展开内容用 `<div>` 包裹：

```html
<details>
<summary>💬 点击展开 · 描述内容（Q1~Q3）</summary>
<div style="color: #2e7d32; font-size: 0.85em;">

**Q1：问题标题？**

回答内容...

**Q2：问题标题？**

回答内容...

</div>
</details>
```

### 样式要求
- 折叠标题格式：`🔻 学习批注 | 描述内容（Q1~Q3）💬`
- 行首 `🔻` 下三角醒目标识，`学习批注 |` 竖线分隔，行尾 `💬` 对话气泡收尾
- 展开内容颜色：深绿色 `#2e7d32`
- 展开内容字号：`0.85em`（比正文小一档）
- `<div>` 开标签后必须空一行，否则 Markdown 加粗等格式不会渲染
- `</div>` 前也空一行

### 编号规则
- 正文序号使用带圈数字：①②③④
- 批注编号使用 Q 系列：Q1、Q2、Q3...（与正文序号区分）
- 批注标题用加粗普通文本 `**Q1：...**`，不用 Markdown 标题（`####`）

### 不需要的元素
- 批注块上方不加独立的 `### 📝 深度批注：xxx` 标题，折叠块的 summary 已经足够
- 不使用 `>` 引用块格式（颜色不可控），统一用 `<div style>` 控制

## 正文格式规范

### 表格
- 对比类内容优先使用 Markdown 表格
- 表头居中对齐 `|:---:|`

### 比喻和类比
- 用生活化的比喻解释技术概念，适合小白阅读
- 比喻前加 emoji 标识（如 🍜、🥩、🍳、📚）

### 代码块
- 用 ``` 围栏代码块展示示例
- 伪代码和示意图用纯文本代码块（不指定语言）
- 实际代码指定语言（如 ```python）

### 文档元信息
- 文档开头包含：作者、邮箱、创建时间、内容来源（如有）
- 文档末尾包含参考来源链接

## Markdown 兼容性注意事项

- `style` 属性在部分渲染器（VS Code 预览、GitHub）中不生效，但不会报错
- `<details>` + `<summary>` 折叠块兼容性好，几乎所有渲染器都支持
- 不依赖 `<font color>` 标签（兼容性差）
- 标准 Markdown 没有颜色语法，需要颜色区分时使用 HTML `<div style>`
